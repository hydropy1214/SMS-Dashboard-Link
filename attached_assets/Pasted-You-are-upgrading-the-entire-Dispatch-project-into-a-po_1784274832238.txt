You are upgrading the entire Dispatch project into a polished, production-quality self-hosted messaging platform. This is a complete architecture refactor, not a phased migration. Replace the existing implementation where necessary while preserving the overall purpose of the project. The final result should feel like a professional SaaS product rather than a prototype.

The core concept remains the same:

A user opens the Dispatch web dashboard, pastes phone numbers, writes a message, clicks Send, and the message is delivered through the user's own Mac using Messages.app and AppleScript. The system must continue to be completely self-hosted with no third-party messaging APIs and no per-message costs.

The project should be redesigned around reliability, maintainability, and scalability while keeping the user experience extremely simple.

The backend should become the central coordinator. It should manage all data, monitor connected Macs, maintain message history, store configuration, and communicate with Mac Agents. The backend should never contain AppleScript logic. Only the Mac Agent should interact with Messages.app.

The Mac Agent should become a standalone application instead of a single script. Organize it into clear modules such as server, sender, AppleScript execution, settings, heartbeat, device detection, logging, health checks, and utilities. Every responsibility should have its own file so the project remains easy to extend.

The Mac Agent should automatically:

* Detect whether Messages.app is installed.
* Detect whether Messages.app is currently running.
* Detect whether AppleScript automation permissions are available.
* Detect available iMessage accounts.
* Detect available SMS forwarding devices.
* Detect macOS version.
* Detect Node.js version.
* Detect CPU usage.
* Detect memory usage.
* Detect uptime.
* Detect hostname.
* Generate a unique Agent ID during first startup.
* Store configuration locally.
* Automatically reconnect if the backend becomes unavailable.
* Retry failed requests using exponential backoff.
* Keep detailed local logs.

The backend should continuously know the status of every connected Mac.
